无聊星人的戈德堡机械

Posted on 2018-01-29 Edited on 2018-10-01

人类到底能有多无聊呢？

先看一个视频。

OK Go - This Too Shall Pass - Rube Goldberg Machine - Official Video

鲁布·戈德堡机械（Rube Goldberg machine），一种被设计得过度复杂的机械组合，以迂回曲折的方法（通常包括连锁反应）去完成一些其实是非常简单的工作，例如倒一杯茶，或打一颗蛋等等。

设计者必须计算精确，令机械的每个部件都能够准确发挥功用，因为任何一个环节出错，都极有可能令原定的任务不能达成。

由于鲁布·戈德堡机械运作繁复而费时，而且以简陋的零件组合而成，所以整个过程往往会给人荒谬、滑稽的感觉。

美国漫画家鲁布·戈德堡在他的作品中创作出这种机械，于是人们就以“鲁布·戈德堡机械”命名这种装置。

在英语世界的用法

“鲁布·戈德堡机械”这个词语首先录于《韦伯斯特第三版新国际词典》，其意思是“以极为繁复而迂回的方法去完成实际上或看起来可以容易做到事情”。

在 1930 年代的美国，鲁布·戈德堡机械被形容为“荒谬地组合在一起的机械”。

自此，“鲁布·戈德堡机械”的意思亦扩延为“极为混乱或复杂的系统”，例如在英语世界里，会有“退休保险有如鲁布·戈德堡机械（一样繁复）”这种说法。

大众文化中的鲁布·戈德堡机械

  • 日本电视剧《33分侦探》，每一集的开头，在侦探家必定有复杂的机器来完成泡咖啡给客人的任务。
  • 鲁布·戈德堡机械曾屡次出现于《X档案》其中一集《哥德堡变奏》中。
  • 电视剧《迷失》角色之一约翰·洛克曾向一名小孩子展示一台小型的鲁布·戈德堡机械。
  • 在电影《死神来了》系列，主角们均是由于日常各种细微的物件互相影响下触发连锁反应而遇害，他们可以说是活生生的身处于一台庞大的鲁布·戈德堡机械中。
  • 在电影《回到未来》，布朗博士用了一个类似鲁布·戈德堡机械的装置去为他的狗造早餐。
  • 日本 NHK 儿童电视节目 ピタゴラスイッチ（毕达哥拉斯的知识开关） 的开始结束以及中间部分皆穿插了类似此种的鲁布·戈德堡机械，用以呈现标题。
  • 动画片《猫和老鼠》中《Designs on Jerry》一集中出现类似鲁布·戈德堡机械。
  • 2005 年，探索频道节目《流言终结者》在《佳节特辑》中制作了一个鲁布·戈德堡机械以庆祝圣诞节。
  • 2010 年，美国乐团《OK Go》，在单曲《This Too Shall Pass》的音乐录影带中，以两层楼的鲁布·戈德堡机械装置完成近 4 分钟一镜到底的影片。
  • 2012 年，美剧《基本演绎法》的片头使用了鲁布·戈德堡机械装置。

软件包管理工具 Homebrew

Posted on 2018-01-28 Edited on 2018-10-01

上一篇 macOS 相关的文章似乎还挺多人读的，于是再来一篇。
然后感觉自己在离开普通读者的道路上眼看是越走越远了。

介绍 Homebrew

了解过 Linux 的同学，一定对 Linux 世界五花八门又便捷强大的软件包管理工具印象深刻。

这么好的东西，macOS 上有没有呢？

答案不言而喻。

Homebrew（简称：brew） 是如今 macOS 最为主流的软件包管理工具，让用户在 macOS 上拥有类似 apt-get 的能力，用以简化软件的安装、管理和卸载过程。

Homebrew 使用 Ruby 进行开发，开源并托管于 GitHub，通过用户的贡献扩大对软件包的支持。如今，绝大多数主流软件和工具都支持 Homebrew。

其实，Homebrew 这个名字直译过来就是“家酿”，有在家自酿啤酒、自产自用的意思，由该项目的发起者马克斯·霍威尔 (Max Howell) 所起。

这里有个题外话，早期计算机历史上曾有一个影响深远的爱好者组织“家酿计算机俱乐部（Homebrew Computer Club）”，史蒂夫·沃兹尼亚克（Steve Wozniak）就是它的成员。

Homebrew 和 MacPorts

除了 Homebrew，macOS 上还有另一个历史更悠久的软件包管理工具，MacPorts，两者的工作方式都是下载源代码，然后在本地编译。

不过，它们的差异也同样不小，主要有以下三点：

  • MacPorts 的理念是尽量减少对系统现有库的依赖，编译时间较长；而 Homebrew 则是尽量依赖系统现有库，编译时间显著减少。
  • MacPorts 将软件是安装到 /opt/local ，不会与系统现有的软件发生冲突；而 Homebrew 则安装到 /usr/local，可能与系统自带的软件发生冲突。
  • Macports 使用 rsync 或 svn 进行同步；而 Homebrew 则使用 git。

总的来说，Homebrew 使用简单，编译时间短，比较适合新手使用。MacPorts 编译时间长，命令还要带上 sudo，易用性上没有 Homebrew 好，但是比较干净，适合有洁癖的人使用。

当然，今天我们要重点介绍的，还是 Homebrew。

应运而生的 Homebrew Cask

Homebrew 使得 Power User 在 macOS 下安装管理 Git、Mysql 等常用工具不再困难。那么，是否也可以通过这样简单的方式来安装 Google Chrome 浏览器呢？

Homebrew-Cask 便是对于 Homebrew 的扩展，有了它，就可以采用 brew 的方式安装图形界面的软件，例如 QQ、 Google Chrome、Evernote 等。

Cask 会从镜像源下载已经编译好了的应用软件二进制包（.dmg/.pkg）到本地，然后解压到单独的目录 /opt/homebrew-cask/Caskroom，并软链（symlink）到 /Applications 目录下。

macOS 软件安装包 pkg 和 dmg

Posted on 2018-01-26 Edited on 2018-11-05

从 Snow Leopard，具体来说，是从当时还被称为 Mac OS X，版本号为 10.6.6 的系统版本，苹果开始为 macOS 提供统一的应用商店，Mac App Store。

也是从那个时候开始，macOS 越来越多地得到，原本一母同胞的小兄弟，iOS 的特性。

不过，相比封闭而年轻的 iOS，macOS 毕竟是更为自由、强大，也更有历史的桌面操作系统。更何况 Mac App Store 自身的体验也相当一般，毛病众多，所以，在 macOS 上，传统的软件安装方式仍然被广泛使用中。

具体来说，多数软件会提供 dmg 或 pkg 格式的安装包。

dmg

dmg 是苹果的压缩镜像文件，类似于 Windows 下的 ISO。

它是 macOS 应用软件通用的打包格式，里面一般包含“应用程序名.app”的图标和一个应用程序文件夹（/Applications）快捷方式。

直接将图标拖曳至应用程序文件夹，即可完成安装。

想要卸载应用，也非常简单绿色，多数时候直接在 Launchpad 或 Applications 目录下删除应用（文件夹）即可。

pkg

pkg 属于系统级软件的安装程序，相当于 iOS 越狱后装的 deb，一般会修改系统配置，权限较高，类似 Windows 下的安装程序 Setup.exe 和 *.msi。

pkg 安装一般要求 sudo 授权，卸载 pkg 安装的应用也比较麻烦。

对于这种软件，除了其自身可能提供的卸载程序，我们还可以借助第三方软件，如 CleanMyMac、AppCleaner 等来彻底卸载删除。

mpkg

pkg 是单个应用程序的安装包，而 mpkg（multi pkg）则是多个 pkg 。

最经常见到的是 macOS 自身的安装器文件，如果你打开从 Mac App Store 下载的“macOS High Sierra.app”，就可以看到一个 .mpkg 文件，其中有着系统安装包主体的文件。

结语

除了使用 dmg、pkg 格式的软件包来安装软件，作为与 *UNX 有着深厚历史渊源，并且拥有众多程序员用户的操作系统，macOS 下也有着优秀的软件包管理工具 —— MacPorts 和 Homebrew。

通过它们，专业用户可以方便地下载、安装和管理丰富的开源软件。

科学的煲鸡汤：一个月一次的奇迹

Posted on 2018-01-20 Edited on 2018-10-01

我们的生活真的就这么平淡无奇么？

前段时间，查找关于「巴德尔-迈因霍夫现象」的资料时， 发现了这个有那么点儿意思的说法。

利特尔伍德定律（Littlewood’s law），指的是一个人大致一个月就会有一次见到所谓的“奇迹”。

来源

这一定律是由英国剑桥大学教授约翰·伊登斯尔·利特尔伍德提出，发表在其 1986 年出版的作品集《一个数学家的杂集》里。

他提出这个定律的目的是破除人们对超自然现象的迷思。

利特尔伍德定律和大数定律相关。

而大数定律认为，只要样本数量足够大，不管多么难以想象的事情都会发生。

这里需要指出的是，这两者都不是经过严格证明的统计学定律。

定律内容

利特尔伍德定义“奇迹”为以百万分之一概率发生的稀有事件。

他假设一个人每天有 8 个小时的清醒时间，在这段是时间内，假设其每秒能觉察到一个事件（看到、听到、碰到，甚至想到……）。

那么，只要 998……不，只要 35 天（35d * 8h * 60min * 60s = 1,008,000s），这个人经历的事件就会到达一百万件。

哪怕去掉那些占据绝大多数的、平常的、重复的事件，出现一个相对稀有的“奇迹”，完全是合理的。

唔……从这个角度来看，我们以为的奇迹确实并非那么罕见。

从 Nintendo Labo 到蒙特梭利

Posted on 2018-01-19 Edited on 2018-11-05

Nintendo Labo

经过一段充满节奏地宣传铺垫，任天堂终于在 1 月 18 日公布了 Nintendo Switch 的新玩法—— Nintendo Labo。

这是一项利用自行组合的瓦楞纸套件和 Nintendo Switch 组合成如钢琴、鱼竿、赛车、枪械和机器人等等各种玩具的全新功能。

Make、Play、Discover，任天堂还是一路既往的那个任天堂。

虽然难免还是有人着眼于硬纸板和价格，不过，这次发布在国内所激发的讨论热度还是超乎了我的预料，甚至还连获得了 CCTV-2 的报道。

不过，在这次讨论中，引起我注意的是一个词儿：

蒙特梭利。

蒙特梭利教育法

蒙特梭利，一种由意大利心理学家兼教育学家玛丽亚·蒙特梭利发展起来的教育方法，出自其出版的《运用于儿童之家的科学教育方法》一书。

蒙氏教育法的特色在于强调独立，有限度的自由和对孩子天然的心理、生理及社会性发展的尊重。

作为一种经过充分实践和发展的教育理论，蒙特梭利自有其详实具体的范式。

不过，我认为，自己查到了两句话足以让不明真相的大家感受一二：

  • 「Mom, help me do it myself」，妈妈，帮帮我，让我能自己做到；
  • 「Never help a child with a task at which he feels he can succeed」，如果孩子觉得这是他自己能完成的事情，就千万不要帮他做。

不论如何，本学渣表示又学习到了。

一点感想

有别于索尼和微软，乃至于世界上的绝大多数游戏厂商，任天堂的游戏便是把玩家当作一个真正的孩子，为你提供的，也是一个个充满乐趣的、拥有探索感的场景或者玩具。

所以，注定任天堂会赢得人们的喜爱，但却无法适应那些自认为已经长大，再也没有童心、没有耐性的人。

事实上，我想到了一个影响力同样十分巨大，并且理念类似的游戏，那就是——我的世界（Minecraft）。

今天就到这儿吧。
