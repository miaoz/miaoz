简易科学上网指南

Posted on 2020-02-28

如何科学上网？
经常被问到，于是写在这。
现在直接用 VPN 的很少了吧，流量特征太明显，比较流行的是 Shadowsocks、V2Ray。
不用担心，本文不会论及原理（超出本人能力范畴），甚至不会讲到具体操作，仅仅只会概述一二方案。

技术人员

如果动手能力比较强，精力也比较充沛。
可以自己搭服务，只要有一台国外的 VPS，其他的都不是问题。

如果能搭起来，理论上这一方案的稳定性是最高的，毕竟只有自己使用，除非遇到大范围抽风的不可抗力。
当然，如果你所使用的 VPS 搭梯子的人多了，也有 IP 段直接被封的风险。

参考：
https://github.com/haoel/haoel.github.io
https://www.hyperapp.fun/zh/Store/shadowsocks-libev.html

轻度折腾

如果既想要专业一点，又不想自己折腾，比较现成的做法是：
购买别人的服务，俗称机场，然后再搭配使用付费或免费的客户端。
这也是我现在主力使用的方案。

推荐的机场：

  • RixCloud：http://rixcloud.com
  • 魅影极速：http://maying.co

推荐的客户端：

  • Surge（付费，Mac、iOS）：https://nssurge.com
  • Shadowrocket（付费，iOS）
  • ShadowsocksR（免费，Win、Android）

注意，如果手机是 iPhone，会需要一个非国区的 Apple ID。

这一方案应该是目前最主流的，通常机场都会至少提供数十个接入点，速度、稳定性和抗风险能力也比较有保障，但是机场的坑也不浅，如果能上外网，推荐参考「毒药机场」评测。

参考：
毒药机场：https://www.duyaoss.com

不折腾

确实有看起来完全不折腾的方案。
比如 ExpressVPN、多态……之流，直接使用服务提供者封装好的软件应用，只要付费就可以了，但是服务的可靠程度存疑。
由于我已经很久没有使用过这一方案，因此不多加讨论。

其他方案

使用海外电话卡，每月数百不等。

推荐：
美国电话卡：Google Fi，须使用 Google Pixel 激活（买一个或借一个），每个月 20 + 刀；
香港电话卡：3HK esim，比较便宜，详情看官网。

这个方案应该是稳定性最高的，除非掀桌子，但是手机流量显然成本不低，可以作为备胎使用。

参考：
https://fi.google.com/about/
https://www.three.com.hk/eSIMMall/chi/eSIMMall/index.jsp

最后

没有一劳永逸，面对真实世界吧。

AC66U B1 折腾：Merlin 改版固件 + 手动安装 Shadowsocks

Posted on 2018-10-01 Edited on 2020-02-28

10.2 更新：在 Github 发现了一个最新版本 SS 的库，这就省事儿了。
下载地址

年纪越大，就越不爱折腾，买了 AC66U B1 后就一直在用官方固件。

后来搬回家，直接用着弟弟留下来的极路由，AC66U B1 束之高阁。

这次导火索是新买了 Google Pixel 2 XL 的“已连接，但无法使用互联网”顽疾，导致不能更新 Android 9，查了下需要有一个能直接爬墙的 WIFI。

再次就是为了让家人能用上 Telegram 的尝试。

总之，我终于还是要补上这落下许久的功课。

刷入 Merlin 固件

根据官方论坛提示，下载并刷入 kooshare 改版梅林固件。

这一步没什么可说的。

刷完后，在【系统管理】 – 【系统设置】内打开 Format JFFS partition at next boot（下次重启格式化 jffs 分区） 和 Enable JFFS custom scripts and configs（开启 jffs 自定义脚本）这两个选项并重启就好。

手动安装 shadowsocks

官方软件中心已经没有了我们需要的东西，所以需要自己动手将 shadowsocks 上传到路由器。

这里有两种方法。

用 ssh 直接链接路由器，在路由器上下载是最简单的。

wget -N -O shadowsocks.tar.gz https://github.com/harry3633/fancyss/raw/master/fancyss_arm/shadowsocks.tar.gz

（这里这个下载地址可能会失效，同时软件的版本可能并不新。）

第二种方法则是下载到本地，然后再上传到路由器，这就要需用到 scp 协议，Mac、Linux 直接使用命令行，Windows 可用 winscp 搞定。

scp /path/filename username@servername:/path

之后 cd 到软件包所在目录，用几行命令一路解压缩、修改权限、执行安装脚本、重启路由器。

一切顺利的话这是最简单的，不过这里我因为各种原因，绕了一点远路。

先是安装了一个旧版本的包，而后发现怎么都搞不定。

接着又通过离线安装功能更新了一个新下载的包，这才装上能用的版本。

最后

整个过程其实不难，难点在于因为众所周知的原因，我们不能便利地获得可信并且足够新的软件包。我在网上翻找的时候，看到了很多过期的信息，要甄别这些只能依靠耐心。

当然，最后的结果不坏。

Android 9 更新了，Telegram 也顺利的注册和使用。

自由的空气自此在这个家庭之中弥漫，哪怕他们也许并不会在意。

但，我的感觉因此很好。

Mac 初上手（5）：通过多种特性与 iOS 协作

Posted on 2018-03-08 Edited on 2018-11-05

今天要说的是 Mac 和 iOS 设备的协作。

如果你在使用 iPhone 或 iPad，那么你的 Mac 将有机会发挥更强大的能力。

如今我们越来越多的使用智能手机，但是一些更为专业的工作，还是无法抛开电脑。

迥异于微软的做法，苹果并不认同简单地将 macOS 与 iOS 融合为一体，但这些年他们还是将不少心思花在了两个系统的配合工作上。

熟练掌握这些特性，不论工作还是娱乐，你都将如虎添翼。

隔空投送（AirDrop）

通过隔空投送，也就是 AirDrop，你可以与附近的其他 Apple 设备，包括 Mac、iPhone、iPad 等，即时地共享照片、视频、文稿等内容。

只要你的设备支持并且打开 AirDrop，同时又开启了 Wi-Fi 和蓝牙，那么通过它传输文件还是很便利的，速度也还很快。

具体怎么使用呢？

你可以打开 Finder（访达）菜单栏上的 AirDrop 窗口，上面会显示附近同样打开了 AirDrop 功能的可用设备。

然后，拖拽想要分享的内容到相应的设备图标上就可以了。

除此之外，你也可以直接在想要分享的文件上，通过「右键菜单 - 分享 - AirDrop」 来进行共享。

相应的，iOS 设备上同样可以使用 AirDrop，只要打开相应的功能，在想要文件的文件里就可以看到附近的设备，点击即可分享。

接力（Handoff ）

通过 Handoff 功能，你可以在一台设备上（Mac、iOS 设备或 Apple Watch）开始一项任务，然后在另一台设备上继续该任务，而不丢失任务进程。

比如，你在路上用 iPhone 上浏览了某个网页，中途没看完就到家了，这个时候打开 Mac，会看到 Dock 左侧多出来一个特殊的 Safari 图标，点击就会打开浏览器，然后从你离开的位置继续浏览。

反过来呢，也是一样的。

大多数苹果官方的应用都可以使用 Handoff 功能，如备忘录、日历、通讯录、邮件、Safari、iWork 套件等等。只要经过适配，第三方应用也可以做到。

只要你的设备登录到同一个 iCloud 帐户，一切便会无缝进行。

是不是很爽？

当然，你要先确认自己的 Mac 和 iOS 设备已经打开了 Handoff 功能。

Mac 在「系统偏好设置 - 通用」里：

而 iPhone 则跟 AirDrop 一样，都在在「设置 - 通用」的相关选项里。

连续互通

除了 AirDrop、Handoff，macOS 和 iOS 之间还有很多美妙的协作特性，苹果将这些特性统一称作连续互通。

比如，那个许多人都听过的笑话：在家里复制，在公司粘贴……这如今在 macOS 和 iOS 之间是可行的。

而如果你的 iPhone 上电话响了，你可以选择在 Mac 上接起来。

甚至你 iPhone 上的短信都可以同步到 Mac 上，并且你也可以在 Mac 上方便地发送和回复。

还有，如果你拥有 Apple Watch，当你靠近 Mac 的时候，可以自动将其解锁。

……

总结

苹果确实没有将 macOS 和 iOS 简单粗暴的融合，但是不知不觉间，他们有许多特性却是围绕着两者的协作。

如果你同时拥有 Mac 和 iOS 设备，那么这些功能真的都非常棒，它们帮助你在多个设备之间自如地切换和协作，让你的工作和娱乐始终处于一种行云流水的状态中。

不过，相比其他科技公司和互联网巨头，苹果的软件质量更不受信任，尤其这几年，我多次看到过有些人说 AirDrop、Handoff 等新功能无法使用。

就我个人而言，虽然多数时候这些特性的使用体验都还不错的，但确实也会偶尔莫名的失效。

我想，这可能就是创新的代价。

但我还是相信，只要苹果能够持续改善可用性，它们带来的便利真的无以伦比！

什么，你说蓝牙耗电不想？

哥，现在都 8102 年了！

Mac 初上手（4）：安装与卸载软件

Posted on 2018-03-05 Edited on 2018-11-05

在此之前，我已经写过一篇关于 macOS 软件安装包的文章，有兴趣的同学可以回去看看。

今天的话题，可以说是那篇文章的一个补充，但要更基础得多。

在 macOS 上，如何安装和卸载软件呢？

简单

首先说一个讲究人应该都知道的 tip。

不论使用 macOS 抑或 Windows，都尽量从官方渠道下载软件，除非万不得已，不要去各种搜索出来的不明小网站。

只要你能做到这一点，电脑的安全性就大大增加了。

具体到 macOS，你可以从 Mac App Store 或者官网下载或者购买软件，当然，也可以通过 Homebrew Cask（我之前也有一篇相关文章）。

如果是从 App Store 下载，就如同 iOS 一样，开盖即食，没什么可说的了。

而如果通过其他方式下载，你通常会得到一个 pkg 或 dmg 的安装包，具体可以看旧文，这里也不细说了。

macOS 软件的格式扩展名是 .app，网站下载的 .app 文件直接拖拽到「应用程序」文件夹就算安装成功了。

一般情况下，卸载软件也很简单。

如果是 App Store 下载的软件，在 Launchpad 按住图标直至抖动，图标左上方就会出现一个小叉，点击即可选择删除，这个逻辑是跟 iOS 一样的。

另外，你也打开「应用程序」文件夹，把想要卸载的软件 ⌘ + delete 键删除或拖拽到废纸篓，然后潇洒地清空废纸篓。

搞定。

真的卸载干净了么？

多数小软件按照以上的方式处理已经足够了。

macOS 的操作逻辑确实很优雅，很多时候你只需要拖动就够了，并不需要理解这些操作后面的细节。

不过，这个世界通常不允许你过得这么轻松。

尤其如果你还有一些洁癖的话。

像 MicroSoft Office、Adobe 系列软件这种庞然大物，如果你要卸载它们，只是拖入废纸篓然后清空，仍然会残留很多文件。

这个时候你有两种办法。

一是这些大型软件官方通常会提供专用的卸载工具，有时候甚至就集成在安装包里，不然官网也会提供，动手找找看吧。

二则是借助第三方工具，如 CleanMyMac、AppCleaner 等，通过它们你终于可以将不受欢迎的家伙们请出家门，还你一个干干净净的 Mac。

还有另一种更恶劣的情况，那就是流氓软件，它们一旦进来，就会使尽一切办法留在你的电脑上。

MacKeeper 就是其中尤其臭名昭著的一个，它的推广力度之大，以至于你可能会在各个地方与之不期而遇，然后不小心中了招，从此如狗皮膏药一样死死粘在你的 Mac 里。

为此，有人针对它写了专杀工具，这待遇可真是绝了！

最后

是不是有点惊讶呢？

不要奇怪，Mac 不是神器，自然需要一定的维护，而 macOS 的世界也不是天堂，有好人也会有坏人。

Mac 的软硬件确实很优秀，使用它确实可以使你少一些对方琐碎的问题，但相应的，Mac 也自有其局限。

在一个流氓横行的地方，每一个流氓看起来都很平常，人们见怪不怪，对付他们也更有经验，而倘若一个地方流氓比较少，那么任何一个流氓看起来都要面目狰狞得多，并且，人们对付它的经验也更少。

嗯，感觉这话可以放在很多地方都挺合适的呢。

把话收回来，安全这种事情，重要的还是养成正确的习惯吧。

Mac 初上手（3）：Multi-Touch 触控板手势

Posted on 2018-03-04 Edited on 2018-11-05

如同 Thinkpad 经典的小红点一样，Mac 的触控板是一大特色。

必须要强调，Mac 上的触控板跟某些 Windows 笔记本上的触摸板完全不是一个等级的存在。

点、按、扫、滚动，一指、两指、三指、四指。

不同的组合之下，只要不是要求达到极度的精确（做设计或玩游戏），普通用户完全可以抛弃鼠标，曾经我的巅峰操作是用它玩「暗黑破坏神2」！

花 10 分钟了解这些手势，可以大大提高 Mac 使用效率和方便性。

Multi-Touch 的很多手势比较像 iPhone 上的操作，比如点击某东西只要轻点触控板即可，而不用完全按下。

比如双指从触控板右侧边缘向左轻扫可查看通知中心，类似 iPhone 上从屏幕上方外侧向内扫；比如双指开合缩放图片和网页，跟 iPhone 上一样。

例：

（双指开合缩放网页或图片等）

（双指置于触控板右侧边缘并向中心拖移以显示通知中心。）

还有更多：


除了 MacBook 配备了触控板，你也可以购买苹果单独出售的 Magic Trackpad 2。

此外，苹果自家是鼠标 Magic Mouse，也继承了一部分手势。

总结

具体想要了解和学习这些手势操作，可以打开「系统偏好设置-触控板」，那里有你想要的一切。
